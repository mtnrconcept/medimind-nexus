1. VUE D'ENSEMBLE DU PROJET
================================================================================

NOM DU PROJET : MediMind Nexus
TYPE : Application web médicale de gestion de patients
OBJECTIF : Plateforme complète de suivi médical avec IA intégrée

DESCRIPTION :
MediMind Nexus est une application web médicale avancée qui combine :
- Gestion complète des dossiers patients
- Visualisation 3D du corps humain (Digital Twin)
- Analyse IA des données médicales
- Système d'alertes médicales intelligent
- Graphiques d'évolution de la santé
- Assistant IA conversationnel (RAG)
- Analyse croisée des données médicales mondiales

PUBLIC CIBLE :
- Médecins et professionnels de santé
- Hôpitaux et cliniques
- Centres de recherche médicale

2. ARCHITECTURE TECHNIQUE
================================================================================

ARCHITECTURE : Single Page Application (SPA)
PATTERN : Component-Based Architecture
RENDU : Client-Side Rendering (CSR)
ÉTAT : React Hooks + Context API

STRUCTURE EN COUCHES :
┌─────────────────────────────────────────┐
│         INTERFACE UTILISATEUR           │
│  (React Components + Tailwind CSS)     │
├─────────────────────────────────────────┤
│         LOGIQUE MÉTIER                  │
│  (Hooks, Services, Utilities)           │
├─────────────────────────────────────────┤
│         GESTION D'ÉTAT                  │

├─────────────────────────────────────────┤

│         BASE DE DONNÉES                 │

└─────────────────────────────────────────┘

3. STACK TECHNOLOGIQUE
================================================================================

FRONTEND :
----------
- React 18.3.1 : Framework UI principal
- TypeScript : Typage statique
- Vite : Build tool et dev server ultra-rapide
- React Router DOM 6.x : Routing et navigation
- TailwindCSS 3.x : Framework CSS utility-first
- Shadcn/ui : Composants UI réutilisables

VISUALISATION 3D :
------------------
- Three.js : Moteur 3D WebGL
- React Three Fiber : Intégration React pour Three.js
- @react-three/drei : Helpers et composants 3D

GRAPHIQUES & CHARTS :
---------------------
- Recharts : Bibliothèque de graphiques React
- Charts pour : IMC, poids, glycémie, tension, maladies

BACKEND & BASE DE DONNÉES :
---------------------------
- Supabase : Backend-as-a-Service
  - PostgreSQL : Base de données relationnelle
  - Edge Functions : Serverless functions
  - Authentication : Gestion des utilisateurs
  - Storage : Stockage de fichiers
  - Real-time : Subscriptions temps réel

INTELLIGENCE ARTIFICIELLE :
---------------------------
- OpenAI API : Modèles GPT pour l'analyse
- RAG (Retrieval-Augmented Generation) : Assistant médical
- Streaming API : Réponses en temps réel
- Analyse prédictive : Recommandations personnalisées

ÉTAT & DONNÉES :
----------------
- React Query (@tanstack/react-query) : Cache et synchronisation
- React Context : État global
- Local Storage : Persistance locale

UI/UX :
-------
- Radix UI : Composants accessibles headless
- Lucide React : Icônes modernes
- Sonner : Notifications toast
- Framer Motion : Animations fluides

UTILITAIRES :
-------------
- clsx / tailwind-merge : Gestion des classes CSS
- date-fns : Manipulation de dates
- zod : Validation de schémas

================================================================================

5.1 COMPOSANTS D'AUTHENTIFICATION

5.2 COMPOSANTS DE LAYOUT
-------------------------

- PageTransition : Animations de transition entre pages

5.3 COMPOSANTS PATIENT (Détail)
--------------------------------

A) AIAssistant
--------------
DESCRIPTION : Assistant IA conversationnel avec RAG
FONCTIONNALITÉS :
  - Chat en temps réel avec streaming
  - Questions suggérées contextuelles
  - Analyse des données patient
  - Historique de conversation
TECHNOLOGIES : OpenAI API, Streaming, Supabase Edge Functions

B) AIPredictionsCard
--------------------
DESCRIPTION : Analyse IA des risques et recommandations
FONCTIONNALITÉS :
  - Détection hypoglycémie/hyperglycémie
  - Analyse des pathologies (diabète, HTA, goutte)
  - Recommandations par catégorie :
    * Médicaments
    * Exercices
    * Nutrition
    * Surveillance
    * Mode de vie
  - Score de risque dynamique
  - Tri par urgence (1-10)
LOGIQUE : Règles médicales basées sur les données biologiques

C) CrossDataAnalyzer
--------------------
DESCRIPTION : Analyse croisée avec données mondiales
FONCTIONNALITÉS :
  - Chargement de 10,000+ pathologies
  - Chargement de 1,800+ symptômes
  - Chargement de 1,000+ traitements
  - Chargement de 6,700+ médicaments
  - Analyse de cohérence
  - Détection d'anomalies
  - Statistiques comparatives
ANIMATIONS : Loader matriciel style Matrix

D) DigitalTwin3DViewer
----------------------
DESCRIPTION : Visualisation 3D du corps humain
FONCTIONNALITÉS :
  - Modèle 3D anatomique interactif
  - Calques anatomiques (peau, tissus, organes, os)
  - Marqueurs d'alertes sur organes affectés
  - Rotation et zoom
  - Vue face/profil/dos
TECHNOLOGIES : Three.js, React Three Fiber, @react-three/drei

E) PatientHealthCharts
----------------------
DESCRIPTION : Graphiques d'évolution de la santé
ONGLETS :
  1. IMC : Évolution avec zones de référence
  2. Poids : Courbe de poids sur 12 mois
  3. Constantes : Tension artérielle
  4. Glycémie : Avec seuils hypo/hyperglycémie
  5. Maladies : Timeline avec chandelles de sévérité
TECHNOLOGIES : Recharts, visualisations interactives

F) PharmacologyMatrix
---------------------
DESCRIPTION : Matrice des interactions médicamenteuses
FONCTIONNALITÉS :
  - Analyse du traitement actuel
  - Détection d'interactions
  - Alertes de contre-indications
  - Effets secondaires

G) SafetyAlertBanner
--------------------
DESCRIPTION : Bannière d'alertes de sécurité
TYPES D'ALERTES :
  - CRITICAL : Urgence vitale (rouge)
  - WARNING : Attention requise (orange)
  - INFO : Information (bleu)

H) VitalSignsPanel
------------------
DESCRIPTION : Panneau des signes vitaux
AFFICHAGE :
  - Tension artérielle
  - Température
  - Glycémie
  - SpO2
  - IMC
  - Fréquence cardiaque

I) ExtendedLabResults
---------------------
DESCRIPTION : Résultats de laboratoire étendus
CATÉGORIES :
  - Hématologie
  - Fonction rénale
  - Bilan lipidique
  - Fonction hépatique
  - Marqueurs inflammatoires
  - Fonction respiratoire
  - Allergies
  - Vitamines/Os
  - Thyroïde
  - Marqueurs cardiaques
  - Électrolytes

J) MedicalHistory
-----------------
DESCRIPTION : Historique médical du patient
AFFICHAGE :
  - Antécédents personnels
  - Antécédents familiaux
  - Allergies
  - Chirurgies passées

6. FONCTIONNALITÉS CLÉS
================================================================================

6.1 GESTION DES PATIENTS
-------------------------
- Liste complète des patients
- Recherche et filtrage
- Fiche patient détaillée
- Données démographiques
- Résultats biologiques
- Notes médicales NLP
- Traitement actuel
- Pathologies associées

6.2 SYSTÈME D'ALERTES INTELLIGENT
----------------------------------
TYPES D'ALERTES :
  - Valeurs biologiques anormales
  - Interactions médicamenteuses
  - Contre-indications
  - Complications potentielles
  - Surveillance requise

DÉTECTIONS AUTOMATIQUES :
  - Hypoglycémie (< 70 mg/dL) - CRITIQUE
  - Hyperglycémie (> 180 mg/dL)
  - Hypertension (> 140/90 mmHg)
  - Insuffisance rénale (DFG < 60)
  - Hyperkaliémie (K+ > 5.5 mEq/L)
  - Anémie (Hb < 10 g/dL)
  - Hypoxémie (SpO2 < 94%)
  - Cytolyse hépatique (ALAT > 50 U/L)
  - Syndrome inflammatoire (CRP > 20 mg/L)

INTERACTIONS MÉDICAMENTEUSES :
  - Metformine + Produit de contraste iodé
  - IEC + Potassium
  - Aspirine + Anticoagulants
  - Méthotrexate + AINS
  - IEC + Toux sèche
  - Statines + Myalgies
  - Corticoïdes + Diabète

6.3 ANALYSE IA PERSONNALISÉE
-----------------------------
ANALYSE DES PATHOLOGIES :
  - Diabète : Surveillance HbA1c, complications
  - HTA : Auto-mesure, réduction sel
  - Goutte : Conseils nutritionnels
  - Insuffisance rénale : Adaptation doses

RECOMMANDATIONS PAR CATÉGORIE :
  - Médicaments : Ajustements, changements
  - Exercices : Activité physique adaptée
  - Nutrition : Aliments à favoriser/éviter
  - Surveillance : Bilans et contrôles
  - Mode de vie : Modifications comportementales

SCORE DE RISQUE :
  - Base : 30 points
  - Hypoglycémie : +35 points
  - Insuffisance rénale : +20 points
  - HTA non contrôlée : +20 points
  - Hyperglycémie : +15 points
  - Diabète : +15 points
  - HTA : +10 points
  - Effets secondaires : +10 points
  - Goutte : +5 points

6.4 ASSISTANT IA (RAG)
----------------------
CAPACITÉS :
  - Analyse contextuelle des données patient
  - Réponses en temps réel (streaming)
  - Questions suggérées intelligentes
  - Historique de conversation
  - Accès aux données médicales mondiales

QUESTIONS TYPES :
  - "Le traitement est-il adapté à cette glycémie ?"
  - "Y a-t-il des symptômes neurologiques ?"
  - "Quels sont les effets secondaires possibles ?"
  - "Compare ce profil avec les données mondiales"

7. BASE DE DONNÉES ET API
================================================================================

7.1 SUPABASE CONFIGURATION
---------------------------
URL : Configurée via VITE_SUPABASE_URL
ANON KEY : Configurée via VITE_SUPABASE_ANON_KEY

7.2 TABLES PRINCIPALES
----------------------

TABLE : patients
COLONNES :
  - id : UUID (PK)
  - patient_id : TEXT (unique)
  - age : INTEGER
  - gender : TEXT
  - nationality : TEXT
  - treatment : TEXT
  - medical_notes_nlp : TEXT
  - lab_results_json : JSONB
  - outcome : TEXT
  - height_cm : NUMERIC
  - weight_kg : NUMERIC
  - created_at : TIMESTAMP
  - pathology_id : UUID (FK)

TABLE : pathologies
COLONNES :
  - id : UUID (PK)
  - name : TEXT
  - icd_code : TEXT
  - category : TEXT
  - description : TEXT
  - symptoms : TEXT[]
  - risk_factors : TEXT[]

7.3 EDGE FUNCTIONS
------------------

FONCTION : ai-assistant
ENDPOINT : /functions/v1/ai-assistant
MÉTHODE : POST
PAYLOAD :
  {
    "message": "Question de l'utilisateur",
    "patient": { données patient },
    "conversationHistory": [ historique ]
  }
RÉPONSE : Server-Sent Events (SSE) streaming
MODÈLE IA : OpenAI GPT (configurable)

7.4 TYPES DE DONNÉES ÉTENDUS
-----------------------------

ExtendedLabResults :
  - Vitaux de base : glucose, tension, température
  - Hématologie : hémoglobine, plaquettes, leucocytes
  - Fonction rénale : créatinine, DFG
  - Bilan lipidique : cholestérol total/HDL/LDL, triglycérides
  - Fonction hépatique : ALAT, ASAT, bilirubine
  - Inflammatoire : CRP, VS
  - Respiratoire : spirométrie, SpO2, FeNO
  - Allergies : IgE, éosinophiles
  - Vitamines/Os : vitamine D, calcium, densité osseuse
  - Thyroïde : TSH, T4
  - Cardiaque : troponine, BNP
  - Électrolytes : sodium, potassium, chlorure

10. GUIDE DE DÉVELOPPEMENT
================================================================================

10.1 INSTALLATION
-----------------
1. Cloner le repository
2. Installer les dépendances : npm install
3. Configurer les variables d'environnement :
   - VITE_SUPABASE_URL
   - VITE_SUPABASE_ANON_KEY
4. Lancer le serveur de dev : npm run dev

10.2 COMMANDES PRINCIPALES
---------------------------
npm run dev          # Démarrer le serveur de développement
npm run build        # Build de production
npm run preview      # Prévisualiser le build
npm run lint         # Linter le code

------------------------
- TypeScript strict mode
- Composants fonctionnels avec hooks
- Props typées avec interfaces
- Nommage : PascalCase pour composants, camelCase pour fonctions
