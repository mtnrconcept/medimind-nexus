1.2 FLUX DE DONNÉES PRINCIPAL
------------------------------

SCÉNARIO : Affichage d'une fiche patient

1. NAVIGATION
   User clique sur patient → React Router change l'URL
   → PatientDetail component se monte

2. CHARGEMENT DES DONNÉES
   useEffect() se déclenche
   → Appel Supabase : supabase.from('patients').select(...)
   → Données récupérées depuis PostgreSQL
   → État local mis à jour : setPatient(data)

3. CALCUL DES ALERTES
   Hook usePatientAlerts() s'exécute
   → Analyse des résultats biologiques
   → Détection des interactions médicamenteuses
   → Génération des alertes (CRITICAL, WARNING, INFO)
   → Retour du tableau d'alertes

4. RENDU DES COMPOSANTS
   Composants enfants reçoivent les props
   → VitalSignsPanel : Affiche les constantes
   → AIPredictionsCard : Génère les recommandations
   → DigitalTwin3DViewer : Initialise le modèle 3D
   → PatientHealthCharts : Génère les graphiques
   → AIAssistant : Prêt pour les questions

5. INTERACTIONS UTILISATEUR
   User pose une question à l'assistant
   → Appel Edge Function : /functions/v1/ai-assistant
   → Streaming de la réponse OpenAI
   → Mise à jour progressive de l'UI

1.3 PATTERN DE COMMUNICATION
-----------------------------

A) COMMUNICATION PARENT-ENFANT (Props)
   PatientDetail (parent)
   ↓ [props: patient, alerts, labResults]
   AIPredictionsCard (enfant)

B) COMMUNICATION ENFANT-PARENT (Callbacks)
   AIAssistant (enfant)
   ↑ [callback: onQuestionAsked]
   PatientDetail (parent)

C) COMMUNICATION GLOBALE (Context)
   AuthProvider (Context)
   → Tous les composants peuvent accéder à l'état auth
   → useAuth() hook pour consommer le contexte

D) COMMUNICATION SERVEUR (API)
   Component
   → fetch() ou Supabase client
   → Edge Function ou Database
   ← Réponse JSON ou Stream

2.1 CYCLE DE VIE D'UNE PAGE PATIENT
------------------------------------

PHASE 1 : INITIALISATION
-------------------------
1. URL change : /patients/:id
2. React Router monte <PatientDetail>
3. États initiaux :
   - patient: null
   - loading: true
   - decrypting: true

PHASE 2 : DÉCRYPTAGE (Simulation UX)
-------------------------------------
1. Animation de décryptage (2 secondes)
2. Affichage :
   - Icône cadenas
   - Barre de progression
   - Message "Décryptage des données..."
3. Transition vers chargement réel

PHASE 3 : CHARGEMENT DES DONNÉES
---------------------------------
1. Appel Supabase :
   ```typescript
   const { data } = await supabase
     .from('patients')
     .select(`*, pathologies (id, name, icd_code, category)`)
     .eq('id', id)
     .maybeSingle();
   ```

2. Parsing des données :
   - lab_results_json → ExtendedLabResults
   - pathologies → Pathology object
   - Conversion des types

3. Mise à jour de l'état :
   ```typescript
   setPatient({
     id: data.id,
     patient_id: data.patient_id,
     age: data.age,
     // ... autres champs
   });
   setLoading(false);
   ```

PHASE 4 : CALCUL DES ALERTES
-----------------------------
1. Hook usePatientAlerts() s'exécute automatiquement
2. Analyse séquentielle :
   a) Interactions médicamenteuses
   b) Contre-indications allergiques
   c) Valeurs biologiques anormales
   d) Surveillance requise

3. Génération des alertes :
   ```typescript
   alerts.push({
     id: 'glucose-hypo',
     level: 'CRITICAL',
     type: 'ABNORMAL_VALUE',
     title: 'Hypoglycémie',
     description: 'Glycémie à 55 mg/dL...',
     organ: 'pancreas',
     action: 'Resucrage immédiat...'
   });
   ```

4. Tri par sévérité :
   CRITICAL → WARNING → INFO

PHASE 5 : RENDU INITIAL
------------------------
1. Layout principal s'affiche
2. Composants se montent dans l'ordre :
   - SafetyAlertBanner (si alertes critiques)
   - VitalSignsPanel
   - DigitalTwin3DViewer
   - PatientHealthCharts
   - AIPredictionsCard
   - AIAssistant

3. Chaque composant :
   - Reçoit ses props
   - Initialise son état local
   - Lance ses effets (useEffect)
   - Rend son UI

PHASE 6 : INTERACTIONS CONTINUES
---------------------------------
1. User interagit avec l'UI
2. Événements déclenchés
3. États mis à jour
4. Re-render des composants concernés
5. Animations et transitions


2.2 GESTION DES ERREURS
------------------------

STRATÉGIE EN CASCADE :

1. TRY-CATCH au niveau composant
   ```typescript
   try {
     const data = await fetchData();
   } catch (err) {
     setError(err.message);
     toast.error('Erreur de chargement');
   }
   ```

2. ÉTATS D'ERREUR
   - error: string | null
   - Affichage conditionnel d'un message
   - Bouton de retry si applicable

3. FALLBACKS
   - Données par défaut si échec
   - UI dégradée mais fonctionnelle
   - Messages utilisateur clairs

4. LOGGING
   - console.error() pour le debug
   - Envoi à un service de monitoring (future)

│  └───────────────────────────────────┘  │
│  ┌───────────────────────────────────┐  │
│  │   Logique                         │  │
│  │   - streamChat()                  │  │

│  └───────────────────────────────────┘  │
└─────────────────────────────────────────┘

┌─────────────────────────────────────────┐
│    Supabase Edge Function               │
│    /functions/v1/ai-assistant           │
└─────────────────────────────────────────┘

┌─────────────────────────────────────────┐

│    (Streaming Response)                 │
└─────────────────────────────────────────┘

FLUX DÉTAILLÉ D'UNE QUESTION :

1. USER TAPE UNE QUESTION
   - Input onChange met à jour l'état
   - Bouton Send devient actif

2. SOUMISSION
   ```typescript
   const handleSubmit = (e) => {
     e.preventDefault();
     if (!input.trim() || isLoading) return;
     streamChat(input.trim());
   };
   ```

3. PRÉPARATION DE LA REQUÊTE
   ```typescript
   const userMsg = { role: 'user', content: userMessage };
   setMessages(prev => [...prev, userMsg]);
   setInput('');
   setIsLoading(true);
   ```

4. APPEL API
   ```typescript
   const response = await fetch(
     `${SUPABASE_URL}/functions/v1/ai-assistant`,
     {
       method: 'POST',
       headers: {
         'Content-Type': 'application/json',
         'Authorization': `Bearer ${ANON_KEY}`,
       },
       body: JSON.stringify({
         message: userMessage,
         patient: { ...patientData },
         conversationHistory: messages,
       }),
     }
   );
   ```

5. STREAMING DE LA RÉPONSE
   ```typescript
   const reader = response.body.getReader();
   const decoder = new TextDecoder();
   let assistantContent = '';

   // Ajouter message vide
   setMessages(prev => [...prev, { 
     role: 'assistant', 
     content: '' 
   }]);

   while (true) {
     const { done, value } = await reader.read();
     if (done) break;

     const chunk = decoder.decode(value);
     const lines = chunk.split('\n');

     for (const line of lines) {
       if (line.startsWith('data: ')) {
         const data = line.slice(6).trim();
         if (data === '[DONE]') break;

         const parsed = JSON.parse(data);
         const content = parsed.choices?.[0]?.delta?.content;
         
         if (content) {
           assistantContent += content;
           // Mise à jour progressive
           setMessages(prev => {
             const updated = [...prev];
             updated[updated.length - 1] = {
               role: 'assistant',
               content: assistantContent
             };
             return updated;
           });
         }
       }
     }
   }
   ```

6. AFFICHAGE PROGRESSIF
   - Chaque token reçu met à jour l'UI
   - Effet de "typing" en temps réel
   - Scroll automatique vers le bas

7. FIN DU STREAMING
   ```typescript
   setIsLoading(false);
   ```

GESTION DES ERREURS :
- Timeout après 30 secondes
- Retry automatique (1 fois)
- Message d'erreur utilisateur
- Log des erreurs pour debug

```typescript
const generateHistoricalData = (
  currentWeight, 
  currentHeight, 
  age
) => {
  const data = [];
  const monthsBack = 12;
  
  for (let i = monthsBack; i >= 0; i--) {
    const date = new Date();
    date.setMonth(date.getMonth() - i);
    
    // Simulation poids (variation ±5kg)
    const weightVariation = (Math.random() - 0.5) * 10;
    const weight = currentWeight + weightVariation - (i * 0.3);
    
    // Hauteur stable pour adultes
    const height = currentHeight + (Math.random() - 0.5) * 0.5;
    
    // Calcul IMC
    const bmi = weight / ((height / 100) ** 2);
    
    // Simulation glycémie (amélioration dans le temps)
    const glucose = 55 + (Math.random() * 40) + (i * 2);
    
    // Simulation tension
    const systolic = 125 + (Math.random() * 20);
    const diastolic = 80 + (Math.random() * 10);
    
    data.push({
      date: date.toLocaleDateString('fr-FR', {
        month: 'short',
        year: '2-digit'
      }),
      weight: parseFloat(weight.toFixed(1)),
      height: parseFloat(height.toFixed(1)),
      bmi: parseFloat(bmi.toFixed(1)),
      glucose: parseFloat(glucose.toFixed(0)),
      systolic: parseFloat(systolic.toFixed(0)),
      diastolic: parseFloat(diastolic.toFixed(0)),
    });
  }
  
  return data;
};
```

```typescript
  queryKey: ['patient', id],

  staleTime: 5 * 60 * 1000, // 5 minutes
  cacheTime: 30 * 60 * 1000, // 30 minutes
});
```

```typescript
useEffect(() => {
  let isMounted = true;
  
  const fetchData = async () => {
    const data = await api.get();
    if (isMounted) {
      setState(data);
    }

  fetchData();
  
  return () => {
    isMounted = false; // Cleanup
  };
```
